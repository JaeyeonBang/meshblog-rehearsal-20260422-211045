# meshblog

A personal 2nd-brain blog that runs as a fully static site on GitHub Pages. Feed it an Obsidian vault; it produces a site with a knowledge graph, per-note Q&A chips, and shareable OG cards — no server, no database at runtime.

## Quick Start

```bash
git clone <repo> meshblog && cd meshblog
bun install
bun run build:fixture   # zero-key preview (5 canned notes, 30 Q&A cards)
bun run build-all       # full pipeline (requires API keys — see Prerequisites)
bun run preview         # static preview @ http://localhost:4321
```

> **Fixture mode** (`build:fixture`) seeds 5 notes and 30 qa_cards from `test/fixtures/seed.sql` and builds the site — no API keys or `claude` CLI required. Use it for first-clone preview, CI smoke tests, and forks.

## Prerequisites

| Requirement | Purpose | Notes |
| :--- | :--- | :--- |
| Node 20+ | Runtime | `node --version` |
| [Bun](https://bun.sh) | Package manager + runner | `curl -fsSL https://bun.sh/install \| bash` |
| `OPENAI_API_KEY` | Embeddings (`text-embedding-3-small`) | [platform.openai.com/api-keys](https://platform.openai.com/api-keys) |
| `OPENROUTER_API_KEY` | Entity extraction (`gpt-4o-mini`) — optional | [openrouter.ai/keys](https://openrouter.ai/keys) |
| `claude` CLI | Q&A generation via `generate-qa` — optional | Silently skips if missing |

Copy `.env.example` to `.env.local` and fill in the values.

## Pages

| Route | Purpose | React hydration |
| :--- | :--- | :--- |
| `/` | Latest posts feed + global Q&A chips | `QAChips` → `client:visible` |
| `/posts/[slug]` | Post detail: markdown body + per-post Q&A + related notes | `MarkdownView` → `client:load`, `QAChips` → `client:visible` |
| `/notes/[slug]` | Atomic KM note: same layout as posts | `MarkdownView` → `client:load`, `QAChips` → `client:visible` |
| `/graph` | d3-force knowledge graph; mode (note/concept) × level (L1/L2/L3) toggle; deep-link via `?mode=&level=` | `GraphView` → `client:only="react"` |

## Full Pipeline

```bash
bun run build-index     # MD → SQLite: entities, embeddings, concept clusters (.data/index.db)
bun run generate-qa     # 3-tier Q&A via claude CLI (silently skips if binary missing)
bun run export-graph    # note-l{1,2,3}.json + concept-l{1,2,3}.json → public/graph/
bun run build-manifest  # public/notes-manifest.json (graph click → correct route)
bun run build           # astro build → dist/
```

Run steps 1–4 + build in one command: `bun run build-all`.

### Artifact Regeneration

| Change | Re-run |
| :--- | :--- |
| Add or edit a note | `build-index` → `generate-qa` → `export-graph` → `build-manifest` |
| Change entity extraction prompt | `build-index --reset-entities` |
| Change Q&A prompts | `generate-qa --force` |
| Graph layout only stale | `export-graph` → `build-manifest` |
| OG images or RSS stale | `build-og` / `build-rss` independently |

## Deploy

GitHub Pages via `.github/workflows/deploy.yml`. Push to `main` triggers:

1. `build-index` (entity extraction + embeddings) — uses `OPENAI_API_KEY` + `OPENROUTER_API_KEY`
2. `generate-qa` — skipped gracefully if `claude` CLI is unavailable in the runner
3. `export-graph`
4. `build-og` + `build-rss`
5. `astro build` → uploads `./dist` to Pages

**Secrets to configure** in the repo (`Settings → Secrets and variables → Actions`):

| Secret | Required? |
| :--- | :--- |
| `OPENAI_API_KEY` | Required for embeddings |
| `OPENROUTER_API_KEY` | Optional — omit to skip entity extraction; site still builds via `FIXTURE_ONLY` fallback |

If neither key is set, the workflow falls back to fixture mode (no embeddings, no entity extraction). The site still builds and deploys with the last-committed artifacts.

## Design System

Editorial B&W aesthetic — hairlines, hover-invert, mono eyebrows, Fraunces display serif — driven by a single `design.md` at the repo root.

### Pipeline

```
design.md  →  scripts/build-tokens.ts  →  src/styles/tokens.css  →  Astro components
```

Edit `design.md` frontmatter (colors, fonts, scale, motion, radius, shadows) → run `bun run build-tokens` → preview. The file is the single source of truth; `tokens.css` is autogenerated — do not hand-edit it.

### Variants

Three presets live in `design.variants/`. Swap the active theme by copying one to `design.md`:

| ID | Name | Character |
| :--- | :--- | :--- |
| a | Editorial B&W | `#000` / `#fff`, Fraunces, hairlines, hover-invert (default) |
| b | Paper & Ink Warm | OKLCH warm paper (h=85°), indigo accent |
| c | Newspaper Dense | B&W, tighter — 36ch prose, 15px body, 12px base space |

```bash
cp design.variants/b.md design.md && bun run build-tokens && bun run dev
```

| Tier | Count | Location |
| :--- | :--- | :--- |

### Invariants (enforced by `blog-bw-polish` skill)

1. No hex literals outside `src/styles/tokens.css` + `src/styles/fonts.css`
2. Hairlines only — `border: 1px solid` default; `3px` reserved for emphasis
3. Hover-invert — interactive surfaces flip `background/color` on `:hover`
4. Mono eyebrows — `--f-mono` + `0.2em` letter-spacing + uppercase + `var(--ink-3)`
5. One shadow only — `var(--shadow-hard)` on `.cmdk`
6. Radius ≤ 4px except `--r-pill` for `.kbd`

## Cost

Full regen at 21 notes: ~$0.006 (OpenAI embeddings + OpenRouter entity extraction). Claude CLI is free (local session). See [`docs/quota-log.md`](docs/quota-log.md) for extrapolations.

## Security

This is a single-author static site. `rehype-raw` is enabled so authored markdown can include raw HTML (embeds, iframes). **Do not** enable `rehype-raw` for multi-author content without adding `rehype-sanitize`. User-generated content is not supported; all `content/` is treated as trusted.

## Rollback

If a build poisons `.data/index.db`, delete it and re-run `bun run build-index`. If `public/graph/*.json` is corrupt, delete and re-run `bun run export-graph`. Since artifacts are CI-regenerated on every deploy, no `git revert` is needed.

## Troubleshooting

### Missing `OPENAI_API_KEY`
```
Error: OPENAI_API_KEY is not set
Fix: add OPENAI_API_KEY=sk-... to .env.local
```

### `claude` binary not found
```
Error: claude: command not found
Fix: install Claude Code CLI → https://docs.anthropic.com/claude-code
     then: claude --version  (must succeed)
```

### OpenAI rate limit (429)
`build-index` retries with exponential backoff (3 retries, up to 10s). If it still fails, wait 60s and re-run. Already-indexed notes are skipped.

### SQLite busy / locked
Schema uses `PRAGMA journal_mode=WAL`. If you see `SQLITE_BUSY`, ensure no other process has `.data/index.db` open.

### Corrupt or unparseable frontmatter
`build-index` logs the failing file path and continues. Fix the YAML front matter in the offending note, then re-run.

### Empty graph (no nodes/links)
Ensure `content/notes/` contains at least one note with `public: true` in frontmatter. Run `bun run build-index` before `export-graph`.
