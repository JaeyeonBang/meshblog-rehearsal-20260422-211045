## Layout

`src/layouts/Base.astro` wraps every page and provides:

- `<html lang>` â€” inferred from Korean/Latin char density (AF1 heuristic) or overridden per-page
- SEO meta: `<title>`, `<meta name="description">`, `canonical` link
- Open Graph: `og:type`, `og:title`, `og:description`, `og:image`, `og:url`
- Twitter Card: `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`
- Atom feed `<link rel="alternate">` â†’ `/atom.xml`
- Skip link (`#main`) + `<nav aria-label="primary">` + `<main id="main">` landmarks (AF6)

## Known Limitations

- **`FIXTURE_ONLY=1` is dev-only.** The `build:fixture` shortcut seeds canned data and is not suitable for production content. Set proper API keys and run `build-all` for a real site.
- **`getQaForConceptsOfNote` uses inline SQL with a subquery.** The subquery joins `concept_entities` â†’ `note_entities` at build time. For vaults with thousands of notes this could be slow; consider materialising the join in `build-index` if build time exceeds 60 s.
- **No runtime search across all notes.** `QAChips` uses Fuse.js on props passed at build time (page-scoped). Vault-wide full-text search requires a pre-built index (Phase 4+).
- **`generate-qa` requires the `claude` CLI locally.** GitHub Actions runners do not have it installed; the workflow step silently skips Q&A generation in CI. Commit `.data/qa/` JSON to preserve cards across deploys.
- **OG image generation depends on `@resvg/resvg-js`** (native module). Arm64 / musl environments may need a different build. The CI workflow guards the step with `if [ -f scripts/build-og.ts ]` to prevent hard failures.
- **No `404.astro` custom error page** â€” Astro static default applies. Planned for Phase 5.
- **Korean slugs appear percent-encoded in the URL bar** (`%EA%B8%80...`). Browsers decode them for display; copy-paste may reveal the encoded form. This is a known trade-off accepted in Phase 3.
