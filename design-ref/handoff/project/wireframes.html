meshblog — 재설계 와이어프레임

실제 리포(JaeyeonBang/meshblog)의 구조에 맞춤: posts+notes, QAChips, 2축 그래프, Astro islands.

3가지 방향은 graph / post / QAChips가 경험에서 차지하는 비중을 다르게 가져갑니다. 에디터 화면은 없음 — Obsidian vault에서 편집 → 빌드 파이프라인이 가공하는 정적 사이트.

routes: / · /posts/[slug] · /notes/[slug] · /graph QAChips (note / concept / global) graph: note×concept × L1/L2/L3 tokens: #fafafa · #1d4ed8 · dark auto
Direction A

Mesh-first

/에 지식 그래프가 히어로로. 아래에 최신 posts + global QAChips. 탐험형 독자 우선.

  • 홈 = 인터랙티브 mesh canvas (mode/level 토글)
  • 노드 클릭 → posts/notes 페이지로 라우팅
  • Q&A는 그래프 아래 2차 요소
강조: 탐험 / 발견
Direction B

Reader-first

피드 중심. 홈은 최신 posts 카드, 우측에 global QAChips + mini mesh. notes는 별도 섹션으로 분리.

  • 홈 = posts 피드 + QA 사이드
  • 상세 = prose 65ch + 하단 per-page QAChips
  • 긴 글/코드 블록에 최적
강조: 가독성 / 깊이
Direction C

Wiki-dense

posts와 notes를 한 스트림으로 평평하게. kind-badge로만 구분. backlinks와 개념 클러스터가 1급.

  • 홈 = 통합 스트림 + 개념 클라우드
  • 상세 = 좌 TOC / 본문 / 우 backlinks+QA
  • atomic note 많은 vault에 적합
강조: 구조 / 밀도
