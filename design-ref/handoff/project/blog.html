post 2026-04-19 · 7 min · blogging / design

왜 meshblog를 만들었나

지식이란 원래 그물처럼 얽혀 있는데, 대부분의 블로그는 그걸 한 줄로 세웠을 뿐이다.

#blogging #design #knowledge-graph

지식은 목차의 모양이 아니다. 뭔가를 배운다는 건 새 단어를 외우는 게 아니라 기존 단어들 사이에 새 엣지를 긋는 일이다. 그런데 블로그는 이 엣지를 기록할 자리가 없다. 날짜 순으로 쌓이거나, 태그로 느슨히 묶이거나, 잘해야 "관련 글 5개" 정도.

기존 블로그의 문제

Medium, Tistory, Velog — 전부 훌륭하지만 한 가지 가정을 공유한다. 글은 단위다. 제목이 있고, 시작과 끝이 있고, 독립적으로 소비된다. 하지만 내가 어떤 개념을 이해한다는 것은 그 개념이 다른 개념과 어떻게 붙어 있는지를 안다는 뜻이다. 이 관계를 보여주지 못하는 매체는 지식의 얕은 단면만 전할 수 있다.

"A tool for thought가 되려면, 매체 자체가 사고의 구조를 흉내내야 한다." — 아마도 내가 방금 지어낸 말.

연결의 가치

Obsidian을 3년 쓰면서 가장 좋았던 건 backlinks다. 내가 쓴 다른 노트가 이 노트를 참조하고 있다는 걸 볼 때, 나는 이 개념이 내 생각의 어디쯤 있는지를 안다. 이건 위키피디아의 "이 문서를 참조하는 문서" 같은 기능이 아니다. 개인 지식 구조의 지도다.

meshblog는 이 감각을 공개 블로그로 옮기려는 시도다. 글 하나가 섬이 아니라 노드이고, 독자는 제목이 아니라 의미로 항행한다.

어떻게 드러내나

세 가지 레이어로.

  • Wiki-link: 본문 안의 [[note-name]]이 실제 링크가 된다.
  • Backlinks: 모든 글 하단에 "이 글을 가리키는 글"이 나타난다.
  • Graph: 전체 vault를 노드 그래프로 볼 수 있다. 클릭하면 해당 글로.
// 실제 파이프라인
export async function buildIndex() {
  const notes = await readVault()
  for (const n of notes) {
    await extractEntities(n)
    await embed(n)
    await upsertBacklinks(n)
  }
}

LLM-generated Q&A

각 글마다 3-tier FAQ를 미리 생성한다. note-tier는 이 글 안에서만 답하는 질문, concept-tier는 관련 개념 전체, global-tier는 vault 전체에 걸친 질문. 이걸 QAChips라고 부르는데, 독자가 검색 대신 질문할 수 있게 해준다.

왜 지금인가

embedding이 저렴해졌다 (~$0.006/full regen). Astro가 React island를 지원한다. SQLite로 빌드타임 쿼리가 충분하다. 그리고 나는 3년치 노트가 쌓였고, 이걸 그냥 썩히기는 아까웠다.

note 2 min · notes/09 · atomic

knowledge-graphs

노드는 개념이 아니라 "구체적인 메모"다. 엣지는 인용과 동시 출현으로 생긴다.

#graph #knowledge

이 노트는 knowledge-graph를 내가 쓰는 맥락에서 무엇을 뜻하는지 고정해두려고 쓴다. 책에 나오는 formal 정의가 아니라 작업용 정의다.

내가 쓰는 의미

노드는 메모다. 하나의 파일, 하나의 주제. 개념이 아니다. 개념은 여러 메모에 걸쳐 나타나는 패턴이다 (후행적으로 클러스터링으로 찾아낸다).

엣지는 세 가지로 생긴다.

  • explicit: 본문 안의 [[wiki-link]]
  • co-occurrence: 같은 entity가 두 메모에서 함께 언급됨
  • embedding-proximity: cosine > 0.82

왜 노드를 개념이 아니라 메모로 두나

개념을 노드로 두면 "무엇이 한 개념인가"를 먼저 정해야 한다. 이건 blocking 이슈다. 메모를 먼저 쓰고 나중에 개념을 뽑으면, 내가 글을 못 쓰는 이유가 사라진다.

graph.addEdge(noteA, noteB, {
  kind: "embed",
  weight: 0.84
})

참고: pagerank-for-notes, louvain-communities.
